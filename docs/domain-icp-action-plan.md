# Domain and ICP Action Plan

This is the action plan for moving `monday-survival` from the CloudBase default test domain to a custom public WeChat H5 domain.

## Current Test Domain

```txt
https://cloud1-d3g4v0ms8ee56bd94-1434679773.tcloudbaseapp.com
```

Use this only for internal testing. CloudBase documentation recommends using default domains for test environments and binding a custom domain for production.

## Target Outcome

- A custom HTTPS domain opens the game directly in WeChat.
- The page no longer shows the CloudBase default-domain risk warning.
- The domain is ready for optional WeChat JS-SDK share-card customization later.

## Recommended Domain Shape

Choose one short subdomain for this game:

```txt
monday.<your-domain>.com
play.<your-domain>.com
week1.<your-domain>.com
```

Avoid putting the game under a long path unless there is a strong reason. A clean subdomain is easier to scan, share, and configure.

## What The User Needs To Prepare

- [ ] Domain name.
- [ ] Domain owner identity: personal or company.
- [ ] Tencent Cloud account that can complete ICP filing.
- [ ] ICP filing subject materials.
- [ ] A phone number that can receive filing verification messages.
- [ ] Decision on whether this is a personal site or company site.
- [ ] Final subdomain, for example `monday.example.com`.

## CloudBase Setup Steps

Based on CloudBase custom-domain documentation:

1. Finish ICP filing for the custom domain.
2. Prepare an SSL certificate.
3. Open CloudBase HTTP gateway / custom domain settings.
4. Add the custom domain.
5. Choose CloudBase CDN for static hosting.
6. Copy the CNAME value generated by CloudBase.
7. Add a CNAME record at the DNS provider.
8. Wait for DNS propagation.
9. Verify the custom domain can open the static site.

## Verification Commands

Replace `monday.example.com` with the final domain:

```bash
curl -I https://monday.example.com/
nslookup monday.example.com
```

Expected:

- HTTP status is `200`.
- `content-type` is HTML.
- DNS resolves through the CloudBase CNAME path.
- WeChat opens the page without a CloudBase default-domain warning.

## After Domain Works

Run the project checks:

```bash
pnpm test
pnpm build
pnpm wechat:check
```

Deploy:

```bash
tcb hosting deploy dist -e cloud1-d3g4v0ms8ee56bd94
```

Generate a new QR code for the custom domain, not the default CloudBase domain.

## Risks

- ICP filing timing is outside the project codebase.
- Custom-domain DNS propagation can take minutes or longer.
- WeChat share-card customization still requires a JS-SDK signature service.
- Adding JS-SDK before the custom domain is ready creates unnecessary backend work.
