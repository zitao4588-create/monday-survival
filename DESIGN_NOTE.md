# 设计说明：今天你能活过周一吗

## 一句话定位

一个关于周一生存的轻量选择小游戏。玩家在一天中的几个压力节点做选择，平衡能量、心情和得分，尝试撑到下班。

## 目标体验

- 单局 2 到 3 分钟。
- 读起来像一个小段子，但选择结果要有轻策略感。
- 适合手机上快速玩一局，然后把结果分享给朋友。
- 失败、勉强存活、体面下班三种结果都要有情绪反馈。

## 核心循环

1. 展示一个周一事件。
2. 玩家从 3 个选项中选择一个。
3. 选择改变得分、能量和心情。
4. 进入下一个事件。
5. 所有事件结束，或能量/心情见底后结算。

## 当前版本边界

- 不做后端。
- 不做数据库。
- 不做登录。
- 不做排行榜。
- 不做远程埋点。
- 暂时不接 Phaser。

## 内容规则

- 回合、选项和数值变化放在 `src/data/turns.ts`。
- 游戏元信息放在 `src/data/metadata.ts`。
- 结局文案放在 `src/data/results.ts`。
- React 组件只负责展示和交互，不直接塞大量剧情文案。

## 移动端重点

- 重点检查 375px 到 430px 宽度。
- 所有选项按钮都要适合触摸。
- 不依赖鼠标悬停。
- 首屏需要直接露出当前回合和至少一个选项。

## 后续可能扩展

- 增加更多周一事件。
- 增加随机事件。
- 增加结果海报。
- 增加“今日周一人格”一类分享文案。
- 增加本地存档，记录最近几次结果。
